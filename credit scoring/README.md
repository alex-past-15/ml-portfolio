# Кредитный скоринг: прогноз дефолта по кредиту

Проект машинного обучения для **оценки кредитного риска** клиентов
банка.

Цель проекта --- построить модель, которая предсказывает, будет ли
кредит:

-   **Fully Paid** --- погашен полностью\
-   **Charged Off** --- перейдёт в дефолт

Задача формулируется как **бинарная классификация**.

------------------------------------------------------------------------

# Описание задачи

Когда клиент подает заявку на кредит, банк должен оценить вероятность
того, что заемщик не выполнит свои обязательства.

Для этого используются **модели кредитного скоринга**, обученные на
исторических данных о ранее выданных кредитах.

В проекте решается задача построения такой модели на основе информации
о:

-   доходе клиента
-   кредитной истории
-   текущих задолженностях
-   кредитном рейтинге
-   цели кредита

------------------------------------------------------------------------

# Данные

Данные представлены двумя файлами.

### credit_train.csv

-   100 000 записей
-   содержит целевую переменную **Loan Status**

### credit_test.csv

-   10 000 записей
-   используется для финальных предсказаний
-   **не содержит целевую переменную**

------------------------------------------------------------------------

# Целевая переменная

Loan Status\
0 --- Fully Paid\
1 --- Charged Off

------------------------------------------------------------------------

# Основные проблемы в данных

## 1. Некорректные значения суммы кредита

В признаке **Current Loan Amount** встречается значение:

99999999

Это нереалистичное значение и было интерпретировано как пропуск.

### Решение

Значение было заменено на NaN и заполнено медианой.

------------------------------------------------------------------------

## 2. Некорректный масштаб Credit Score

Кредитный рейтинг должен находиться в диапазоне:

0--800

Однако в данных встречались значения больше 7000.

### Решение

Такие значения были исправлены делением на 10.

------------------------------------------------------------------------

## 3. Пропуски в данных

Пропуски обнаружены в нескольких признаках:

-   Credit Score
-   Annual Income
-   Months since last delinquent
-   Years in current job

### Решение

Использованы различные стратегии:

-   медианное заполнение
-   отдельная обработка для некоторых признаков
-   создание индикаторов пропусков

------------------------------------------------------------------------

# Feature Engineering

Для улучшения модели были созданы дополнительные признаки.

### Debt-to-Income

Debt_to_Income = Monthly Debt / Annual Income

### Credit Utilization

credit_utilization = Current Credit Balance / Maximum Open Credit

### Логарифмические признаки

-   log_annual_income
-   log_credit_balance

Эти признаки помогают модели лучше учитывать финансовые соотношения.

------------------------------------------------------------------------

# Использованные модели

Были обучены и сравнены:

-   Logistic Regression
-   Random Forest
-   Gradient Boosting

Основная метрика --- **ROC-AUC**.

------------------------------------------------------------------------

# Лучшая модель

Лучший результат показала модель **Gradient Boosting**.

После подбора гиперпараметров:

ROC-AUC ≈ 0.705

------------------------------------------------------------------------

# Оптимизация порога классификации

По умолчанию используется порог 0.5.

Для кредитного скоринга важнее обнаруживать дефолты, поэтому был выбран:

threshold = 0.2

Это позволило значительно увеличить Recall.

------------------------------------------------------------------------

# Финальные метрики

Accuracy: 0.547\
Precision: 0.308\
Recall: 0.800\
F1-score: 0.444\
ROC-AUC: 0.705

Модель обнаруживает примерно **80% потенциальных дефолтов**.
