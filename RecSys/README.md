# Рекомендательная система фильмов

## Описание проекта

Цель проекта — построить систему рекомендаций фильмов на основе предпочтений пользователей.

В рамках проекта реализованы следующие подходы:
- Baseline (популярные фильмы)
- Content-based filtering (по жанрам)
- Collaborative filtering (по пользователям)
- Матричная факторизация (SVD)

## Данные

Используется датасет MovieLens (ml-latest-small).

Содержит:
- movies.csv — информация о фильмах (название, жанры)
- ratings.csv — оценки пользователей

## Анализ данных (EDA)

В ходе анализа было выявлено:

- Матрица пользователь–фильм сильно разрежена (~98%)
- Пользователи чаще ставят высокие оценки (3–5)
- Большинство пользователей оставляют небольшое количество оценок (long tail)

## Подходы

### Baseline модель

Рекомендация популярных фильмов на основе среднего рейтинга и количества оценок.

Модель не является персонализированной и используется как точка отсчёта.

### Content-based filtering

Рекомендации строятся на основе характеристик фильмов (жанров).

Используется cosine similarity между фильмами. Пользователю предлагаются фильмы, похожие на те, которые он оценил высоко.

### Collaborative Filtering

Рекомендации формируются на основе поведения пользователей.

Строится user-item матрица, затем рассчитывается похожесть пользователей с помощью cosine similarity. Фильмы рекомендуются на основе оценок похожих пользователей.

### Матричная факторизация (SVD)

Используется метод SVD для разложения матрицы пользователь–фильм на скрытые факторы.

Это позволяет учитывать неявные предпочтения пользователей и улучшает качество рекомендаций.

## Метрики

Для оценки модели SVD использовалась метрика RMSE.

RMSE: ~0.88

Дополнительно рассматривалась возможность оценки рекомендаций через Precision@K и Recall@K.

## Выводы

- Baseline модель проста, но не учитывает персональные предпочтения
- Content-based ограничен характеристиками фильмов
- Collaborative filtering учитывает поведение пользователей и даёт более релевантные рекомендации
- SVD показывает наилучшие результаты за счёт использования скрытых факторов

## Ограничения

- Высокая разреженность матрицы
- Проблема cold start
- Смещение в сторону популярных фильмов
